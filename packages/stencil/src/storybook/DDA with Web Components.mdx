# DDA with Web Components

This is a step-by-step guide to use vanilla web components.

### Setup

First, import the package script, replacing `X.X.X` with the
[latest published version](https://www.npmjs.com/package/@dubai-design-system/components-js).
**Always pin an exact version in the CDN URL** — with an
unversioned URL, jsdelivr resolves each file independently, so the loader and its chunks
can come from different package versions, which breaks rendering (components hydrate but
never render, with a "unknown to this Stencil runtime" console error):

```html
<script type="module">
    import { defineCustomElements } from 'https://cdn.jsdelivr.net/npm/@dubai-design-system/components-js@X.X.X/loader/index.es2017.js';
    defineCustomElements();
</script>
```

Next, add the two icon fonts used by DDA (Material Icons and Material Symbols) inside the
`<head>` tag:

```html
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200" rel="stylesheet">
```

### Using with a bundler (Vite, Rollup)

> ⚠️ If you install `@dubai-design-system/components-js` from npm and build with **Vite**
> (or plain Rollup), do **not** use the lazy loader
> (`import { defineCustomElements } from '@dubai-design-system/components-js/loader'`).
> The loader resolves component chunks with computed dynamic imports that these bundlers
> cannot statically analyze, so the chunks are missing from the build output — the page
> requests `dda-*.entry.js` files that 404 and the components never render.

Instead, import the self-registering custom-elements build for each component you use:

```js
// each import automatically defines its custom element
import '@dubai-design-system/components-js/dist/components/dda-button.js';
import '@dubai-design-system/components-js/dist/components/dda-header.js';
```

The CDN `<script>` approach above is unaffected — this only applies to bundled npm installs.
A working Vite example lives in the repository under `samples/js/`.

You should now be able to use DDA components:

```bash
<dda-button 
    button_color="default-primary" 
    start_icon="sentiment_satisfied"
    end_icon="arrow_forward" 
    custom_class="" 
    mode="light" 
    button_id="button" 
    aria_label="button"
    size="lg"
    button_shape="circle">
    Click Here
 </dda-button>
```
